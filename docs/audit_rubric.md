# Audit Rubric

Target score: 85+

| Criterion | Weight |
|---|---:|
| Data authenticity | 15% |
| Data quality | 15% |
| Feature correctness | 15% |
| Signal explainability | 15% |
| Backtest rigor | 15% |
| Software architecture | 10% |
| Documentation | 10% |
| Presentation readiness | 5% |

## Score Bands

| Score | Meaning |
|---:|---|
| Below 60 | No-go |
| 60-74 | Technical prototype |
| 75-84 | Interview-ready |
| 85-92 | Strong portfolio |
| 93+ | Institutional-grade public-data project |

## Minimum Acceptance

The project needs real public data ingestion, raw data metadata, clean staging data, tested feature logic, explainable risk signals, honest backtesting, clear documentation, and one-command reproducibility.
